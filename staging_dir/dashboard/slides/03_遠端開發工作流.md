# 遠端開發工作流

**建立日期：** 2026-03-04　｜　**狀態：** 🟢 已部署

---

## 工具組合概覽

| 工具 | 角色 | 解決的問題 |
|------|------|-----------|
| Tailscale | 安全網路層 | 穿透 NAT／防火牆，不需開 port |
| Mosh | 不斷線 SSH | 網路切換後自動重連，UDP-based |
| Tmux | 持久工作環境 | 斷線後 session 繼續跑，多視窗 |
| Claude Code | AI 直接操作 | 在目標機器上讀寫檔案，無中間層 |

---

## 為什麼要這樣組合

```
沒有這套工具的痛點：
  坐捷運換網路 → SSH 斷線 → 程式停了 → 重新連 → 重新跑
  改 Pi5 的程式 → 複製貼上指令 → 等結果 → 再複製 → 效率低

有了這套之後：
  換網路 → Mosh 自動重連，畫面還在
  斷線 → Tmux session 還活著，attach 回來繼續
  改程式 → Claude Code 直接在 Pi5 上讀寫，沒有中間人
```

## 各工具詳細說明

### Tailscale
- 底層：WireGuard（高效能加密 VPN）
- 裝置之間直接 P2P 連線
- Pi5 Tailscale IP：`100.69.230.13`
- 優點：不需固定 IP、不需 port forwarding、外網任何地方都能連

### Mosh（Mobile Shell）
- 協定：UDP（SSH 是 TCP）
- 網路中斷恢復後自動重連，不需重新登入
- 有 local echo 補償，高延遲打字不卡
- 使用 port：UDP 60000–61000

### Tmux
- 前綴鍵：`Ctrl+a`
- 分割視窗：`Ctrl+a |`（左右）、`Ctrl+a -`（上下）
- 切換 pane：`Alt + 方向鍵`
- Detach：`Ctrl+a d`（session 繼續在背景跑）
- 一鍵進入：`~/work.sh`

### Claude Code on Pi5
- 版本：2.1.66
- 安裝位置：`/usr/local/bin/claude`（symlink → `~/.npm-global/bin/claude`）
- 登入方式：Max 帳號 OAuth（`claude /login`）
- 直接在 Pi5 上操作 Pi5 的檔案，無需 SSH 中間層

## 連線方式對照

| 裝置 | 指令 | 特性 |
|------|------|------|
| Mac | `pi`（alias） | mosh + 自動 attach tmux session |
| 手機 | Termius app | SSH + Tailscale，快速查看 |
| 任何地方 | `mosh pi53@100.69.230.13` | 完整 mosh 連線 |

---

## Claude Code 使用模式

| 模式 | 說明 | 適用場景 |
|------|------|---------|
| **Pi5 直接跑** | SSH 進 Pi5 → `claude` | 修改 Pi5 上的程式（最常用）|
| **Mac 跑** | Mac 終端機 → `claude` | 操作 Mac 本機檔案 |
| **Remote Control** | Mac 跑 `claude remote-control` | 出門後用手機 claude.ai app 繼續 Mac 的任務 |
