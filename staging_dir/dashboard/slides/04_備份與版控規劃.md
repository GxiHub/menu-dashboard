## 完成項目總覽

### Phase 1：基礎設施 ✅

```
• git config（user.name / email / defaultBranch）
• gh CLI 安裝 + 認證
• gh auth setup-git（HTTPS credential 自動處理）
```

### Phase 2：6 個 Repo init + push ✅

每個 repo 都完成：`git init` → `.gitignore` → `git add` → `commit` → `gh repo create --private` → `push`

### Phase 3：.gitignore 策略 ✅

**通用排除：**
```
__pycache__/  *.pyc  .env  *.bak*  venv/  *.db  *.sqlite
```

**Dashboard 專用追加：**
```
releases/  incoming/  _work/  previous/  _prod_backup/
_snapshot/  _handoff/  logs/  .telegram_env  DEPLOYED.txt
```

### Phase 4：機密檔集中管理 ✅

```
~/secrets/
├── .gitignore      ← 內容為 *（防止任何檔案被 git 追蹤）
└── telegram_env    ← TELEGRAM_TOKEN + CHAT_ID

symlink:
  ~/dashboard/.telegram_env → ~/secrets/telegram_env
```

- Docker bind mount 會跟隨 symlink，服務不受影響

### Phase 5：自動備份 ✅

**腳本：** `~/github_backup.sh`

```
• 掃描 6 個 repo
• 有變更 → git add -A → commit → push
• 無變更 → 跳過（不產生空 commit）
• 日誌：~/logs/github_backup.log
```

**Cron 排程：** 每 6 小時自動執行

```
0 */6 * * * bash /home/pi53/github_backup.sh
```

## 機密檔清單

| 檔案 | 集中位置 | symlink | 備份方式 |
|------|---------|---------|---------|
| Telegram Token + Chat ID | `~/secrets/telegram_env` | `~/dashboard/.telegram_env` | GitHub 不追蹤，需手動記錄 |
| Cloudflare Tunnel credential | `/etc/cloudflared/*.json` | （原位）| 需另外備份，無法放 GitHub |
| Cloudflare Tunnel config | `/etc/cloudflared/config.yml` | （原位）| 同上 |

⚠️ **注意：** Cloudflared credential 是唯一無法從 GitHub 復原的項目，建議額外備份至密碼管理器。
