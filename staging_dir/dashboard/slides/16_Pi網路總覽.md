# Pi 網路總覽

**類型：** 系統架構　｜　**版本：** v1.0　｜　**建立：** 2026-03-11

---

## 設計目標

各台 Pi 各司其職，透過 **Tailscale** 組成私有內網，由 **pi53** 作為主控 Dashboard，提供一鍵查詢所有節點狀態的能力。

---

## 節點一覽

| 名稱 | 角色 | 本地 IP | Tailscale IP | 主要服務 |
|------|------|---------|--------------|---------|
| **pi53** | 主控 Dashboard | 192.168.1.111 | 100.69.230.13 | dashboard-v2, menu-dashboard, telegram-bot |
| **pi52** | 煮麵自動化系統 | 192.168.0.21 | 100.98.225.85 | noodle-app:5000 |
| _(future)_ | 待新增 | — | — | — |

## 標準化端點：`/api/pi_info`

每台 Pi 必須實作此端點，回傳以下 JSON：

```json
{
  "name": "pi52",
  "role": "煮麵自動化系統",
  "model": "Raspberry Pi 5",
  "ip": {"local": "192.168.0.21", "tailscale": "100.98.225.85"},
  "ui_url": "http://100.98.225.85:5000",
  "services": [{"name": "noodle-app", "port": 5000, "status": "running"}],
  "sensors": {"pt100": 93.5, "igbt": 62},
  "induction": {"cmd_pwr": 70, "actual_pwr": 38, "error": 0, "auto": true},
  "uptime": "up 2 hours, 30 minutes",
  "updated": "2026-03-11T03:30:00"
}
```

---

## 新增節點流程

1. 在新 Pi 的 web app 加入 `/api/pi_info` 端點
2. 在 pi53 的 `pi_nodes.json` 新增該節點設定
3. pi53 的 Pi 網路頁面自動顯示新節點

---

## 查看方式

**pi53 Dashboard → Pi 網路**（需登入）
