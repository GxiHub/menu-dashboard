# 主理人效率餐飲系統

**Owner Efficiency Restaurant System**

**建立日期：** 2026-03-06　｜　**狀態：** 🟡 規劃中

---

## 系統核心目標

打造一套**可複製的餐飲營運模型**，透過五個核心手段：

1. 單店雙品牌
2. 自動化設備
3. 標準化流程
4. 人力效率設計
5. 全流程數位化（POS → 財務 → CRM → KDS）

達到以下目標：

| 目標 | 說明 |
|------|------|
| 提升單店營收能力 | 同一空間創造兩份收入 |
| 降低人力成本 | 用設備和流程取代人頭 |
| 提高開店成功率 | 驗證過的模型，不靠摸索 |
| 主理人收入翻倍 | **而不是工時翻倍** |

---

## 目標客群：在風險中成長的主理人

**特徵**：
- 想創業或已創業
- 有營運能力但缺乏完整系統
- 不希望高額加盟限制
- 希望提高收入與效率

**主要痛點**：

| 痛點 | 現況 |
|------|------|
| 開店成功率低 | 90% 餐廳 1 年內倒閉 |
| 人力成本過高 | 人事佔營收 25-30%，人力短缺嚴重 |
| 單品牌營收有限 | 店面時段利用率不到一半 |
| 營運依賴個人經驗 | 無法複製、無法休息 |

## 模組一：開店風險降低

> 降低新主理人開店初期的不確定性

### SOP 系統

標準化流程包含：

| 流程 | 內容 |
|------|------|
| 出餐流程 | 備料 → 組裝 → 出餐，每步有標準時間和品質基準 |
| 清潔流程 | 日清 / 週清 / 月清，檢核表管理 |
| 日常營運流程 | 開店準備、交接班、打烊收尾 |
| 設備操作流程 | 每台設備的操作手冊 + 異常處理 |

### 店內動線與物流

| 設計項目 | 目的 |
|---------|------|
| 店內動線 | 減少人員無效走動 |
| 食材流動 | 從冷藏 → 備料區 → 煮台，單向不交叉 |
| 出餐動線 | 完成品最短路徑到客人手上 |
| 備料區設計 | 集中配置，一人可操作多工位 |

### 半自動化設備

設備目的：**降低技術依賴**

- 自動升降煮台 — 不需要師傅控火候
- 半自動出餐設備 — 標準化份量和流程

> 有了 SOP + 動線 + 設備，新主理人不用從零摸索。

## 模組二：營運效率提升

> 提高每位員工的產值

### 人力設計

標準角色定義：

| 角色 | 職責 | 特點 |
|------|------|------|
| 煮台手 | 操作煮台設備、控制出餐節奏 | 設備輔助，低技術門檻 |
| 撿料手 | 備料、配料、食材準備 | SOP 標準化，可快速上手 |
| 收銀手 | 點餐、結帳、簡單外場 | 可由自助點餐部分取代 |

關鍵特點：
- **可輪替** — 每個人都能做每個位置
- **不依賴高技術人員** — 不需要廚師，只需要操作員

### 自動化輔助

| 設備功能 | 效果 |
|---------|------|
| 降低操作難度 | 新人訓練時間從數週縮短到數天 |
| 提高出餐穩定度 | 每次出品品質一致 |
| 降低人力需求 | 同樣營業額，少請 2-3 人 |

### 數位化營運系統（新增）

在設備自動化的基礎上，進一步將**整條出餐流程數位化串接**：

| 系統模組 | 功能 | 帶來的價值 |
|---------|------|-----------|
| POS 收銀台 | 數位點餐，取代紙本/口頭點單 | 下單即計時，零漏單 |
| 財務系統串接 | POS 營收數據直通帳務 | 每日自動對帳，即時掌握損益 |
| CRM 客戶管理 | 搭配客戶電話，累積消費紀錄 | 回購分析、精準行銷推播 |
| 工作站顯示（KDS） | 各站獨立螢幕即時看單 | 不用喊單、不漏單、可追蹤效率 |

### 全流程數位化工作流

從點餐到出餐，每一站都在系統中留下數據：

```
📱 POS 點餐（開始計時）
    │
    │  訂單自動派送
    ▼
📺 撿料站（工作站螢幕）
    │  撿料手看螢幕備料 → 完成按「確認」
    │
    │  訂單自動流轉
    ▼
🍲 滷鍋站（工作站螢幕）
    │  滷鍋手放入自動升降機器 → 滷製完成按「出餐」
    │
    │  系統跳出檢核清單
    ▼
✅ 出餐檢核
    │  逐項確認每項食材是否打包
    │  全數確認 → 完成出餐（停止計時）
    ▼
📊 數據回流 → 主理人 Dashboard
```

**數位化帶來的三層價值：**

| 層面 | 效果 |
|------|------|
| 效率面 | 全流程計時 → 找出瓶頸站點；各站螢幕 → 零喊單零漏單 |
| 經營面 | POS → 財務自動入帳；電話 → CRM 回購分析；出餐檢核 → 降低漏裝客訴 |
| 數據面 | 每筆訂單全程追蹤；各站效率可量化；主理人即時掌握全局 |

> POS 是入口 → 串財務 → 串 CRM → 串工作站，每個環節都產生數據。

## 模組三：單店營收放大

> 提高單店收入，而不是增加店數

### 核心策略：單店雙品牌

同一店面運作兩個品牌，服務不同時段、不同客群：

| | 午餐品牌 | 晚餐品牌 |
|--|---------|---------|
| 品類 | 高翻桌產品（例：水餃） | 高毛利產品（例：滷味） |
| 時段 | 11:00 - 14:00 | 16:00 - 22:00 |
| 客群 | 上班族、快速用餐 | 家庭、朋友聚餐 |
| 特性 | 出餐快、量大 | 客單高、毛利好 |

### 共享資源

兩個品牌共用：
- **店面租金** — 一份租金、兩份營收
- **廚房設備** — 同一套設備做兩種品類
- **人力** — 同一組人、不同時段

### 效果

- 時段利用率從 40% → 80%
- 坪效提升 60-75%
- 外送平台雙品牌曝光 = 兩間店的流量

| 方法 | 說明 |
|------|------|
| 驗證過的營運模型 | 第一間店跑通，數據證明可行 |
| 可標準化的流程 | SOP + 設備 + 人力設計，打包成系統 |
| 降低試錯成本 | 設備租賃、流程現成，失敗損失可控 |

最終形成：**可複製餐飲模型**

```
Phase 1（0-6 月）         Phase 2（6-18 月）        Phase 3（18 月+）
──────────────          ──────────────           ──────────────
單店驗證                  標準化輸出                 規模化
• 測試菜單與流程           • SOP 定版               • 設備租賃包
• 確認損益數據            • 供應鏈建立              • 營運系統授權
• 優化設備配置            • 培訓體系建立            • 更多主理人加入
```

## 商業模式

### 系統提供四件事

1. **自動化設備** — 降低技術門檻
2. **營運流程系統** — SOP + 動線 + 人力設計
3. **數位化營運平台** — POS + 財務 + CRM + KDS 工作站

| 來源 | 說明 |
|------|------|

> **系統收入建立在主理人成功的基礎上。**
>
> 主理人賺錢 → 系統才有收入。利益一致，不是對立。
